Security model

How SecureShield Protects Secrets

  • Text, photos, and videos are encrypted in your browser using AES-GCM before upload.
  • Server stores ciphertext only and never plaintext content.
  • The decryption key is kept in URL fragment (#...) and not sent in HTTP request line.
  • Each share has a generated 6-digit access code that must be entered on reveal page.
  • Secrets auto-expire and are deleted after allowed view count.
  • Security headers and rate limits are enabled by default.

For production, deploy behind HTTPS and place it behind a managed database and WAF.
